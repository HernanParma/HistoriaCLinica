🎯 Problema Identificado

El formulario de nueva consulta no estaba funcionando porque:

  • Faltaba el event listener: No se estaba capturando el evento submit
  • No se obtenía el patientId: La variable no estaba disponible en el scope correcto
  • Faltaba la función de envío: No había lógica para procesar y enviar los datos

💡 Cómo Funciona Ahora

  1. Event Listener: Se captura el evento submit del formulario
  2. Obtención de patientId: Se extrae de la URL usando URLSearchParams
  3. Procesamiento de datos: Se recopilan todos los campos del formulario
  4. Validación: Se verifica que los campos requeridos estén presentes
  5. Envío al backend: Se hace POST a /api/pacientes/{id}/consultas
  6. Manejo de respuesta: Se muestra éxito o error según corresponda

🚀 Para Probar en la Aplicación

  1. Ve a la página de historia clínica de un paciente (ej: historia.html?id=1)
  2. Llena el formulario de nueva consulta con los datos
  3. Haz clic en "Guardar Consulta"
  4. Deberías ver un mensaje de éxito
  5. La consulta aparecerá en "CONSULTAS ANTERIORES"
